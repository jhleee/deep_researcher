## 4. CoVe Plan Node (약 80초)

**입력 초안:**
```
한국의 탄소 배출량은 2023년 기준 약 6.1억 톤이다.
```

```
   자료(예: 환경부, IPCC 등)에서 발표한 정확한 수치인가?

3. 한국의 탄소중립이란 정확히 어떤 정의이며, 현재 정책과 실제
   목표가 합치하는 것인가?
4. 탄소 배출량 산정 시 배출원의 범위(예: CO2 외의 온실 가스)가
   명시되어 있는지 확인해야 하는가?
```

- 수치 정확성, 출처 신뢰도, 정의 명확성 검증 질문 생성

### 관찰 사항

1. **Thinking 모델 특성:** qwen3.5-9b는 `<think>` 태그로 추론 과정을 출력. `ThinkingModelWrapper`가 이를 자동 제거하여 파이프라인과 호환.
2. **Context 제한:** 워커의 누적 findings가 길어지면 context 초과 발생. `max_input_chars=6000` 제한으로 해결.
3. **CoVe 질문 수:** 제한 없으면 13개까지 생성 → 검증기 13회 호출. `max_questions=5` 파라미터로 제어.
4. **Sanitizer 교정 루프:** 실제 LLM이 `[출처: ...]` 형식을 일관되게 사용하지 않아 1차 검증에서 FAKE_CITATION 감지. 교정 후 인용 제거/수정하여 2차 통과.
5. **JSON 구조화 출력:** `_repair_truncated_json()`이 max_tokens로 잘린 JSON을 복구하여 안정성 확보.
